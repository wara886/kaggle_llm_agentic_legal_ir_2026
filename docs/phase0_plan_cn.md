# 阶段 0 执行计划（评测底座 + 最小基线骨架）

## 1. 目标与原则
- 目标：先建立“可提交、可诊断、可回归”的最小闭环。
- 原则：
  - 不污染其他项目；
  - 中文说明优先；
  - 严格区分基线与后续增强实验。

## 2. 交付范围
1. 数据处理底座  
内容：读取 train/val/test/corpora、解析 `gold_citations`、引用规范化、语料主表构建、缺失标签统计。

2. 本地评测底座  
内容：实现 citation-level Macro F1、逐查询评估输出、严格模式与语料感知模式（corpus-aware mode）。

3. 错误分析模板  
内容：生成中文错误分析 Markdown 模板，支持后续人工诊断与误差分类（error taxonomy）。

4. 最小 BM25 基线  
内容：语料加载、BM25 检索、提交文件生成。

## 3. 目录约定
- 代码：`src/legal_ir/`
- 脚本：`scripts/`
- 文档：`docs/`
- 产物：
  - 统计与评测：`artifacts/`
  - 预测与提交：`outputs/`

## 4. 风险控制
- 对 `train` 中语料外标签，评测时输出两套结果：
  - `strict`：严格金标；
  - `corpus_aware`：仅评估语料内可命中的金标。
- 保留缺失标签样本清单，便于后续做 citation normalization 与标签清洗策略。

## 5. 阶段 0 验收标准
- 可运行数据准备脚本并输出缺失标签统计。
- 可运行 baseline 脚本并生成 `submission.csv`。
- 可运行评测脚本并输出：
  - 汇总指标（Macro F1）
  - 逐查询评估文件
  - 错误分析模板文件
